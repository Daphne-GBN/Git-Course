# 06 - Branching & Merging

A branch is a movable, lightweight pointer to a commit - creating one is
instant and copies nothing. Branches let you build a feature in isolation,
without touching `main`, until the work is ready to come back together.

`HEAD` marks wherever you currently are - it moves every time you switch or
commit.

Covers `git branch`, `git switch` / `git checkout`, `git merge`, and
deleting branches with `-d` / `-D`.
